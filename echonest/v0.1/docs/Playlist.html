GETplaylist.static

Returns a static playlist.

Test console
Description
Parameter
string

The callback function for JSONP requests. Required if format is jsonp.

callback

Optional

string

The type of the playlist to be generated.

type

Optional

string

The artist_pick parameter is used to determine how songs are picked for each artist in artist-type playlists. If the asc or desc suffix is ommitted, artist_pick defaults to descending.

artist_pick

Optional

string

The maximum variety of artists to be represented in the playlist. A higher number will allow for more variety in the artists. 0 - 1 (default = 0.3)

variety

Optional

string

Controls the distribution of artists in the playlist. A focused distribution yields a playlist of songs that are tightly clustered around the seeds, whereas a wandering distribution yields a playlist from a broader range of artists.

distribution

Optional

string

Controls the trade between known music and unknown music. A value of 0 means no adventurousness, only known and preferred music will be played. A value of 1 means high adventurousness, mostly unknown music will be played. A value of auto indicates that the adventurousness should be automatically determined based upon the taste profile of the user. This parameter only applies to catalog and catalog-radio type playlists.0 - 1 (default = 0.2)

adventurousness

Optional

string

The desired number of songs in the playlist. 0 < results < 100

results

Optional

string

The maximum tempo for any included songs. 0.0 < tempo < 500.0 (BPM)

max_tempo

Optional

string

The minimun tempo for any included songs. 0.0 < tempo < 500.0 (BPM)

min_tempo

Optional

string

The maximum duration of any song on the playlist. 0.0 < duration < 3600.0 (seconds)

max_duration

min_duration

Optional

string

The maximum loudness of any song on the playlist. -100.0 < loudness < 100.0 (dB)

max_loudness

min_loudness

Optional

string

The maximum energy of any song. 0.0 < energy < 1.0

max_energy

string

The minimum energy of any song. 0.0 < energy < 1.0

min_energy

Optional

string

The maximum artist familiarity for songs in the playlist. 0.0 < familiarity < 1.0

artist_max_familiarity

Optional

artist_min_familiarity

artist_max_hotttnesss

Matches artists that have an earliest start year before the given value. Examples: 1970, 2011, present

artist_start_year_before

artist_start_year_after

artist_end_year_before

Optional

string

Matches artists that have a latest end year after the given value. Examples: 1970, 2011, present

artist_end_year_after

Optional

string

The maximum hotttnesss for songs in the playlist. 0.0 < hotttnesss < 1.0

song_max_hotttnesss

Optional

The minimum hotttnesss for songs in the playlist. 0.0 < hotttnesss < 1.0

The maximum longitude for the location of artists in the playlist. -180.0 < longitude < 180.0

max_longitude

Optional

The minimum longitude for the location of artists in the playlist. -180.0 < longitude < 180.0

min_longitude

Optional

string

The maximum latitude for the location of artists in the playlist. -90.0 < latitude < 90.0

max_latitude

Optional

string

The minimum latitude for the location of artists in the playlist. -90.0 < latitude < 90.0

min_latitude

Optional

string

The mode of songs in the playlist. (minor, major) 0, 1

mode

Optional

string

The key of songs in the playlist. (c, c-sharp, d, e-flat, e, f, f-sharp, g, a-flat, a, b-flat, b) 0 - 11

dmca

Optional

string

The user rating for the previous track. 5 is the highest rating, 1 is the lowest rating. Some playlist algorithms will adapt playlists based upon these ratings.

rating

Optional

string

If true, an additional 'track' is added to the end XSPF playlist that is a URL to the next XSPF in the chain.

chain_xspf

Optional

string

Using the previously played track as the basis, make all future tracks some multiplier of the selected attribute(s). required terms use a multiplier in a boost-like format - e.g. energy^.5 to make all future songs have half the energy of the previously played song, or loudness^1.3 to add 30%.

steer

Optional

string

Using the current playlist as a basis, add or remove terms relating to the artists and songs to be played. Applying a term again will overwrite the previous term, including boost. Boosting with a "0" will remove a term from the list of qualifiers. Using 'description' rather than 'steer_description' will clear the description list and restart the playlist session with the new term.

steer_description

Optional

string

Using the current playlist as a basis, add or remove terms relating to the style of artists and songs to be played. Applying a style again will overwrite the previous style, including boost. Boosting with a "0" will remove a term from the list of qualifiers. Using 'style' rather than 'steer_style' will clear the style list and restart the playlist session with the new style.

steer_style

Optional

string

Using the current playlist as a basis, add or remove terms relating to the mood of artists and songs to be played. Applying a mood again will overwrite the previous mood, including boost. Boosting with a "0" will remove a mood from the list of moods. Using 'mood' rather than 'steer_mood' will clear the mood list and restart the playlist session with the new mood.

steer_mood

Optional

string

Banning will prevent the song/all songs by that artist from appearing again in the current playlist session.

ban

Optional
