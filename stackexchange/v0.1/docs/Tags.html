GETtagList

Returns the tags found on a site.

Default
Description
Parameter
string

Required text in returned tags.

filter

Optional

string

Unix timestamp of the minimum creation date on a returned item.

fromdate

Optional

string

Maximum of the range to include in the response according to the current sort.

max

Optional

string

Minimum of the range to include in the response according to the current sort.

min

Optional

string

How the current sort should be ordered. One of desc (default), or asc.

order

Optional

string

The pagination offset for the current collection. Affected by the specified pagesize.

page

Optional

string

The number of collection results to display during pagination. Should be between 0 and 100 inclusive.

pagesize

Optional

string

The sorts accepted by this method operate on the follow fields of the tag object:

popular – count
activity – the creation_date of the last question asked with the tag
name – name
popular is the default sort.

sort

Optional

string

Unix timestamp of the maximum creation date on a returned item.

todate

Optional

string

inname

Optional

stackoverflow
string

To specified on which site you want to search. stackoverflow (default)

GETtagFaqGet

Returns the frequently asked questions for the given set of tags in {tags}.

Default
Description
Parameter
string

For a question to be returned, it must have all the tags in {tags} and be considered "frequently asked". The exact algorithm for determining whether a question is considered a FAQ is subject to change at any time.

{tags} can contain up to 5 individual tags per request.

GETtagRelatedGet

Returns the tags that are most related to those in {tags}.

Including multiple tags in {tags} is equivalent to asking for "tags related to tag #1 and tag #2" not "tags related to tag #1 or tag #2".

count on tag objects returned is the number of question with that tag that also share all those in {tags}.

{tags} can contain up to 4 individual tags per request.

This method returns a list of tags.
